# Pawdise

A place where lost pets live on.

Upload a photo of your pet, and Pawdise transforms it into pixel art living happily in a cosmic virtual world. Check in anytime to see what your pet is up to — chasing butterflies, napping in a sunny meadow, or exploring the stars.

No chat. No dialogue. Just quietly watching your pet live on.

---

## Demo

> Built in 10 hours at a hackathon.

**Flow:** Upload photo &rarr; AI generates pixel art &rarr; Visit your pet in Pawdise

## Tech Stack

| Layer | Technology |
|-------|-----------|
| **Framework** | Next.js 14 (App Router) |
| **Database & Auth** | Supabase (Postgres, Storage, Google OAuth) |
| **Text AI** | Anthropic Claude (claude-sonnet-4-6) — generates pet activity descriptions |
| **Image AI** | Replicate (nerijs/pixel-art-xl) — generates pixel art from photos |
| **Styling** | Tailwind CSS |
| **Deployment** | Vercel |

## Features

- **Photo to Pixel Art** — Upload a pet photo and get a unique pixel art version generated by AI
- **Living Pet Activities** — Claude generates what your pet is doing right now in Pawdise (observation only, no chat)
- **Emotional Loading** — Warm, personalized messages while your pet's pixel art is being created
- **Cosmic Theme** — Deep purple starfield backgrounds with gold accents and particle animations
- **Guest & Auth Flow** — Create a pet without an account, then claim it later with Google sign-in
- **Multi-layer Fallback** — If pixel art generation fails: auto-retry &rarr; text-only prompt &rarr; species sprite fallback
- **Mobile-first** — Designed for phones, scales up to desktop

## Getting Started

### Prerequisites

- Node.js 18+
- npm
- Accounts for: [Supabase](https://supabase.com), [Anthropic](https://console.anthropic.com), [Replicate](https://replicate.com)

### Setup

1. **Clone the repo**

   ```bash
   git clone https://github.com/your-org/pawdise.git
   cd pawdise
   ```

2. **Install dependencies**

   ```bash
   npm install
   ```

3. **Configure environment variables**

   ```bash
   cp env.example .env.local
   ```

   Fill in your keys in `.env.local`:

   ```
   NEXT_PUBLIC_SUPABASE_URL=
   NEXT_PUBLIC_SUPABASE_ANON_KEY=
   SUPABASE_SERVICE_ROLE_KEY=
   ANTHROPIC_API_KEY=
   REPLICATE_API_TOKEN=
   ```

4. **Set up Supabase**

   - Create a `pets` table (see schema below)
   - Enable Supabase Storage for photo uploads
   - Configure Google OAuth in Auth settings

5. **Run the dev server**

   ```bash
   npm run dev
   ```

   Open [http://localhost:3000](http://localhost:3000)

### Database Schema

The `pets` table in Supabase:

| Column | Type | Description |
|--------|------|-------------|
| `id` | uuid (PK) | Pet ID |
| `name` | text | Pet name |
| `species` | text | cat, dog, bird, rabbit, other |
| `breed` | text | Breed description |
| `traits` | text | Personality traits |
| `habits` | text | Habits and preferences |
| `bio` | text | Owner's personal note |
| `original_photo_url` | text | Uploaded photo URL |
| `pixel_art_url` | text | Generated pixel art URL |
| `replicate_job_id` | text | Replicate prediction ID |
| `current_activity` | text | Latest AI-generated activity |
| `current_scene` | text | Current scene name |
| `last_activity_at` | timestamptz | When activity was last generated |
| `session_id` | uuid | Guest session identifier |
| `user_id` | uuid | Supabase auth user (after claim) |

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/pets` | Create a new pet |
| `GET` | `/api/pets/:id` | Get pet details |
| `GET` | `/api/pets/mine` | Get current user's pets |
| `POST` | `/api/pets/:id/upload-photo` | Upload photo & start pixel art generation |
| `GET` | `/api/pets/:id/pixel-art-status` | Poll pixel art generation status |
| `GET` | `/api/pets/:id/activity` | Generate a new activity for the pet |
| `POST` | `/api/pets/:id/claim` | Associate a guest pet with an authenticated user |
